# Database Development Guide

## Architecture

```
SQL Queries (*.sql)
    ↓ sqlc generate
Go Code (*_sql.go)
    ↓ wails3 generate bindings
TypeScript (queries.ts, models.ts)
```

## Directory Structure

```
internal/database/
├── schema/
│   └── schema.sql              # Complete SQLite schema (72 tables)
├── queries/
│   ├── users.sql               # User queries
│   ├── sessions.sql            # Session queries
│   ├── agents.sql              # Agent queries
│   ├── messages.sql            # Message queries
│   └── ...                     # Other domain queries
├── generated/                  # Generated by sqlc (DO NOT EDIT)
│   ├── models.go               # Type definitions
│   ├── querier.go              # Interface
│   ├── users.sql.go            # User query implementations
│   └── ...
└── db.go                       # Database service

frontend/bindings/              # Generated by Wails (DO NOT EDIT)
└── .../generated/
    ├── models.ts               # TypeScript types
    └── queries.ts              # TypeScript functions
```

## Workflow

### 1. Adding/Updating Queries

Edit SQL files in `internal/database/queries/`:

```sql
-- internal/database/queries/users.sql

-- name: GetUserWithSettings :one
SELECT u.*, s.* 
FROM users u
LEFT JOIN user_settings s ON u.id = s.id
WHERE u.id = ?;
```

### 2. Generate Code

```bash
# Generate both Go and TypeScript code
make generate

# Or separately:
make db-generate          # Generate Go code
make bindings-generate    # Generate TypeScript bindings
```

### 3. Use in Frontend

```typescript
import { GetUser, CreateUser } from '@/bindings/.../queries';
import type { User, CreateUserParams } from '@/bindings/.../models';

// Use the generated functions
const user = await GetUser(userId);

const newUser = await CreateUser({
  id: 'user-123',
  username: 'john',
  email: 'john@example.com',
  // ... other fields
});
```

## sqlc Configuration

Configuration is in `sqlc.yaml`:

```yaml
version: "2"
sql:
  - engine: "sqlite"
    queries: "internal/database/queries"
    schema: "internal/database/schema"
    gen:
      go:
        package: "db"
        out: "internal/database/generated"
```

## Query Naming Convention

sqlc uses special comments to generate code:

- `:one` - Returns single row
- `:many` - Returns multiple rows
- `:exec` - Executes without returning data
- `:execrows` - Returns number of affected rows

Example:
```sql
-- name: GetUser :one
SELECT * FROM users WHERE id = ?;

-- name: ListUsers :many
SELECT * FROM users LIMIT ? OFFSET ?;

-- name: DeleteUser :exec
DELETE FROM users WHERE id = ?;
```

## Type Safety

The entire stack is type-safe:

1. **SQL** → sqlc validates against schema
2. **Go** → sqlc generates type-safe Go code
3. **TypeScript** → Wails generates type-safe TS bindings

If you make a mistake in SQL, sqlc will catch it at generation time!

## Tips

1. **Always regenerate after SQL changes**: `make generate`
2. **Never edit generated files**: They will be overwritten
3. **Use transactions**: Access via `dbService.WithTx()`
4. **Check generated types**: Look at `models.go` and `models.ts`

## Schema Updates

To update the database schema:

1. Edit `internal/database/schema/schema.sql`
2. Run `make generate`
3. Database will be updated on next app start

Note: Schema changes are applied via `database.NewService()` which runs the schema on initialization.
